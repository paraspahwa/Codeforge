---
name: pr-conventions
description: Use when writing PR descriptions, commit summaries, or review notes for this repository.
---

When drafting PR or review text for CodeForge:

- Lead with user-visible outcome, not implementation diary
- List risk + test plan in bullet form
- Keep titles under 72 characters
- Mention migrations, env vars, and compose changes explicitly
- Do not claim features that are not in the diff
